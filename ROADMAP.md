# Drift - Feature Roadmap

## Critical Priority
Core features needed for basic functionality

- [ ] Network communication (WebSocket/WebRTC)
  - [ ] Real-time message synchronization between users
  - [ ] Room/lobby system for joining chat sessions
- [ ] User identification
  - [ ] Username system
  - [ ] User colors/avatars for message differentiation
- [ ] Overlay controls
  - [ ] Hide/show toggle
  - [ ] Minimize functionality
  - [ ] Draggable window positioning

## High Priority
Important features for usability

- [ ] Keybindings
  - [ ] Hotkey to toggle overlay visibility
  - [ ] Hotkey to focus chat input
  - [ ] Configurable keybinds in settings
- [ ] Click-through mode
  - [ ] Allow mouse input to pass through to underlying application
  - [ ] Toggle between interactive and click-through modes
- [ ] Message persistence
  - [ ] Save chat history locally
  - [ ] Load previous messages on startup

## Medium Priority
Enhancements and quality of life features

- [ ] Controller quick access bindings
  - [ ] Controller support for opening/closing overlay
  - [ ] D-pad/button navigation through UI
- [ ] Picture/screenshot support
  - [ ] Share images in chat
  - [ ] Screenshot capture integration
- [ ] Settings panel
  - [ ] Customize overlay appearance (size, transparency, theme)
  - [ ] Configure notification settings
  - [ ] Adjust font size

## Low Priority
Future enhancements and exploratory features

- [ ] Discord/Steam integration
  - [ ] Bridge messages to/from Discord servers
  - [ ] Steam friends integration
- [ ] Server hosting options
  - [ ] Self-hosted server support
  - [ ] Public server discovery
